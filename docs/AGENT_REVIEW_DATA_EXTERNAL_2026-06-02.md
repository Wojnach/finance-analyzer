Every metals_loop restart → `get_ofi_zscore` returns 0.0 until 10 fresh readings re-accumulate, `get_rolling_ofi` 0.0 until 3 snapshots → ~5-10 min of dead/garbage microstructure (OFI z-score, spread z-score, multiscale) feeding `orderbook_flow` after each restart, with no warming-up signal. FIX: persist raw deques (`list(_ofi_history[t])` etc.) and add `load_buffers_on_startup()` re-seeding under `_buffer_lock` when file age < 120s.
- `portfolio/earnings_calendar.py:64,174-177` + `portfolio/alpha_vantage.py:94,150`: **[REPEAT]** P0: AV free-tier throttle uses `{"Information": "...25 requests per day..."}` key, recognized by neither module. `_fetch_earnings_alpha_vantage` → no `quarterlyEarnings` → None → falls to yfinance; if that also None, `get_earnings_proximity` caches `{"data": None}` for the full 24h TTL → **24h cache-of-None**: `should_gate_earnings` returns False all day → BUY signals NOT gated near a real earnings event. FIX: cache only successful results (or short negative-TTL 1-2h); add `"Information"` to AV rate-limit checks at lines 94/150 (return None, don't burn the circuit breaker).
- `data/crypto_data.py:73,102,148` + `portfolio/social_sentiment.py:32,65` + `portfolio/sentiment.py:172-197`: **[REPEAT]** P0: un-throttled raw `requests.get`/`yf.Ticker().news` to Yahoo, CryptoCompare, Alternative.me, Reddit, none routed through `http_retry` or any limiter. `data/crypto_data.py` runs in the metals_loop process while `portfolio/*` run in main loop; `shared_state._RateLimiter` instances are **per-process** in-memory buckets, so across the 5 loop processes (DataLoop, MetalsLoop, CryptoLoop, MstrLoop, OilLoop) there is NO shared budget → concurrent Yahoo hammering → 429/IP ban silently starves every loop of fallback data. FIX: route through `http_retry.fetch_json`, gate behind `_yfinance_limiter`/new shared limiter; back with a file/OS token bucket for cross-process safety.
- `portfolio/sentiment.py:172-197`: **[NEW]** P0: `_fetch_yahoo_headlines` calls `yf.Ticker().news` with NO `yfinance_lock` (module imports it nowhere). `get_sentiment` runs from the 8-worker ticker pool (`signal_engine.py:3492-3503`); BTC and ETH are distinct `_cached` keys, so two workers run yfinance concurrently. FIX: wrap `yf.Ticker().news` + the crypto Yahoo fallback (line 162) in `with shared_state.yfinance_lock:`.
