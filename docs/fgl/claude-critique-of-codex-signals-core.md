# Claude Critique of Codex Review — Signals Core

Headline: Codex's review was generated against the `.worktrees/adv-signals-core` snapshot which intentionally excludes `portfolio/tickers.py` (the home of `SIGNAL_NAMES`), causing one P0 finding (#8 about `btc_proxy`) to misread the runtime registry — the issue was already fixed at HEAD (commit efedf4ee) by adding `btc_proxy` to `SIGNAL_NAMES`. The seven remaining P0 findings about outcome backfill timing/truncation and SQLite-vs-JSONL freshness are accurate and correspond to real, currently-shipping bugs. Codex missed several P1/P2 issues that Claude flagged: `btc_proxy` inflating applicable counts on non-MSTR tickers, `cot_positioning`/`dxy_cross_asset` not in `_METALS_ONLY_SIGNALS`, `_compute_applicable_count` ignoring per-horizon blacklist, and the unanimity/entropy stages reading pre-persistence buy/sell counts (codex found one of these — finding #18 — but missed the related applicable-count and onchain-on-ETH leaks).
