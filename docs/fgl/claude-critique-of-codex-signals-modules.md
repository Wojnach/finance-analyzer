[CONFIRM] portfolio/signals/credit_spread.py:79 — `class _Shim: @staticmethod __call__` is a type-attribute bug. Verified at lines 78-82: `_Shim()` instances do NOT delegate `()` to a `@staticmethod __call__` because Python looks up `__call__` on the type and a `staticmethod` descriptor on the type returns the underlying function (no instance binding). Calling `fetch_with_retry(url, ...)` actually *would* work in CPython 3.10+ because `staticmethod` is callable directly since 3.10 — but the code path is fragile and unidiomatic. Claude flagged P1, true severity is closer to P2/cosmetic on modern Python.
