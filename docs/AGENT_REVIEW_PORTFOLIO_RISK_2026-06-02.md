- `portfolio/exit_optimizer.py:319-339, 555-597, 408-453`: **[NEW]** P0: entire exit engine LONG-only. `_compute_pnl_sek` warrant branch (327-332) uses positive leverage; candidate exits generated only at session-**max** quantiles *above* spot (567 skips `target<=price*0.999`); knock-out override (430-450) assumes `financing_level<price`. - `portfolio/portfolio_mgr.py:35,41-47,211-234` + `warrant_portfolio.py:42-48,199-280` + `trade_guards.py:32,126-127,264-330`: **[REPEAT]** P0: cross-process state corruption. `update_state` holds a **process-local** `threading.Lock` around read-modify-write, but state JSONs are mutated by ≥4 OS processes (main loop, Layer 2 subprocess, dashboard, metals/bigbet/iskbets). `atomic_write_json` prevents torn files but NOT lost updates: A reads, B reads+writes, A writes → B's trade silently overwritten. `save_warrant_state` doesn't even take the process-local lock.
