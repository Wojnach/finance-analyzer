## Patterns

1. **Module-level mutable state with ad-hoc locking.** `agent_invocation` has 12+ globals, only some protected by `_completion_lock`. Health and reporting follow the same pattern with `_health_lock` / `_module_lock`. Encapsulating in dataclasses with explicit ownership would catch many P1s here.

2. **Fail-open on parse/IO error.** Multiple paths (`_no_position_skip`, `_load_guard_warnings`, claude_gate config check, signal accuracy lookups) fail OPEN — proceed with default. For safety-critical paths like drawdown checks the code correctly fails CLOSED. The inconsistency makes it hard to reason about safety properties; document a per-gate matrix.

3. **Auth-failure detection has accumulated 4 layers of defense.** Markers, line-start check, line-limit, fenced-code-block tracking. The complexity reflects the 3-week silent outage. But each layer increases the chance of false-negatives if the CLI changes its output format. A more robust approach: parse the structured JSON output (`--output-format json`) and check for `error.code == AUTH_FAILURE` once the CLI exposes that, instead of string matching.

4. **Time semantics inconsistent.** Monotonic clock for agent elapsed, wall clock for `last_trigger_time`, ISO strings for heartbeats. Re-parse/clock-skew bugs cluster around the boundary (BUG-203, BUG-P2B, clock-skew flip cooldown). A single time-source policy would help.

5. **Cycle counter vs wall clock.** Safeguard intervals, log rotation, IC refresh all use cycle counts that assume a fixed 60s or 600s cadence. After the 2026-04-09 cadence bump, several of these became 10x less frequent than documented. Audit all `% N` time gates.

6. **Hardcoded module-level paths defeat xdist isolation.** `journal.py`, `trigger.py`, `agent_invocation.py`, `reporting.py`, `health.py` all hardcode `DATA_DIR / "xxx.json"` at module level. CLAUDE.md flags this; the rule isn't followed in orchestration code.

7. **Tier classification is brittle to string changes.** trigger.py uses regex on reason strings to decide downshift, and `_extract_ticker` uses regex on the same strings. Any rename of "consensus" → "Consensus" or addition of new actions silently breaks routing. Use structured reason objects instead.
