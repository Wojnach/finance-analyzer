## Mechanism

- **Review surface:** isolated worktree `Q:/fa-rev-0531` @ branch
  `review/snapshot-2026-05-31` (off `main` HEAD `9ecdeecc`). Live loops keep
  running off `Q:/finance-analyzer` — worktree never touched by them.
- **Empty baseline:** orphan branch `review/baseline-empty` (empty tree).
  `git -C /q/fa-rev-0531 diff review/baseline-empty -- <path>` renders any file
  as a pure-addition diff so diff-oriented reviewers have full content.
- **8 fresh review subagents** spawned in background, one per subsystem, each
  briefed with the project-specific failure history (not generic smells).
- **Independent pass (me):** I review the 3 highest-blast-radius subsystems
  myself in parallel with the agents, so synthesis can cross-check agent claims
  against a second reading rather than trusting them blind.
- **Cross-critique:** agent findings vs my findings — confirm / refute /
  dedup. Refuted = likely hallucination, demoted with reasoning.
- **Synthesis:** `docs/reviews/2026-05-31-SYNTHESIS.md`, severity-tagged.

## Why no code-change premortem

/fgl step 3 premortems a *code change*'s incident vectors. This session ships
**zero production code** — only review docs. The analogous risk is **wrong or
misleading findings** (false positives that waste a future session, or false
negatives that miss a real bug). Mitigations baked into the flow:
- independent second reading of the riskiest subsystems (catches agent FPs),
- explicit cross-critique phase that must mark each finding confirmed/refuted,
- severity discipline: P0/P1 require a concrete causal chain + file:line, not
  "could break".

## Severity legend
- **P0** correctness/safety bug that can lose money, corrupt state, or silently
  stall the loop. Concrete causal chain required.
- **P1** wrong-but-recoverable: bad signal/accuracy math, missing error handling
  on a real failure path, atomicity gap.
- **P2** robustness / maintainability / latent footgun.
- **P3** style / nit.
