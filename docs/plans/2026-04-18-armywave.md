## Stream 2 — 12 critical-errors entries (Layer 2 contract noise)

**Root cause (verified via direct log inspection, not the triage agent's
hypothesis):**

Overnight, Layer 2 invocations were **`skipped_offhours`** by design — a
legitimate non-run state. Example from `data/invocations.jsonl`:

```
22:24:42  reasons=["XAU-USD consensus BUY (80%)"], status=skipped_offhours
22:42:43  reasons=["XAU-USD flipped BUY->HOLD"], status=skipped_offhours
23:12:49  reasons=["ETH-USD flipped SELL->HOLD"], status=skipped_offhours
...
```

But `check_layer2_journal_activity()` in `portfolio/loop_contract.py` only
treats `status == "invoked"` as non-silent (precondition 4 — the in-flight
suppression). It doesn't know about `skipped_offhours` / `skipped_busy` /
`skipped_gate` / `skipped_stack_overflow`, so it fires violations for every
trigger that fires while Layer 2 is legitimately skipped.

Compounding: **no violation deduplication**. When the same trigger
fires violations every cycle (e.g., trigger 22:42:43 fires
violations at 22:53, 23:03 — same trigger, age +10m each cycle), the
journal gets spammed with duplicates.

**Fix (minimal, additive):**

1. In `check_layer2_journal_activity`, extend precondition 4 to also
   suppress on any `status.startswith("skipped_")`. A skipped invocation
   is a correct non-run — there's no journal-write obligation.
2. Add a precondition 5 (violation dedup): track the last
   `(trigger_ts, violation_fired_ts)` in `contract_state.json`. If the
   current trigger_ts matches the one we already fired for, skip. Reset
   when a new journal entry is written (so a genuinely new silent
   episode for the same trigger still fires once).

**Resolution strategy:**

All 12 entries will be resolved via a single batch resolution append
documenting the pattern — they're all symptoms of the same two bugs.
