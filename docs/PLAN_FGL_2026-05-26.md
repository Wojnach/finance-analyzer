## Protocol

1. Spawn 8 reviewer subagents in **single message** (parallel background):
   - `caveman:cavecrew-reviewer` for narrow subsystems (avanza-api, infrastructure)
   - `pr-review-toolkit:code-reviewer` for broader scope (signals-core, signals-modules,
     orchestration, portfolio-risk, metals-core, data-external)
   - Each writes findings to `docs/AGENT_REVIEW_<SUBSYSTEM>.md`
   - Severity format: `path:line: <severity>: <problem>. <fix>.`
   - Severities: P0 (broken/data loss/correctness), P1 (likely-incident), P2 (latent risk),
     P3 (style/nit — usually skip)
2. While subagents run, write own pass `docs/ADVERSARIAL_REVIEW_INDEPENDENT_2026-05-26.md`
   touching ≥1 finding per subsystem.
3. Collect subagent outputs. Read each.
4. Cross-critique: which subagent missed what own pass caught? Which own-pass findings
   does subagent contradict? Note disagreements.
5. Synthesis doc `docs/ADVERSARIAL_SYNTHESIS_2026-05-26.md`:
   - Dedup'd P0/P1/P2 ranked
   - Top 10 highest-impact actionable items
   - Tie back to historical incidents (auth outage, BUG-97, etc.)
6. Commit each doc separately on `fgl/2026-05-26` branch.
7. Merge to main: `git checkout main && git merge --no-ff fgl/2026-05-26`.
8. Push: `cmd.exe /c "cd /d Q:\finance-analyzer && git push"`.
9. Cleanup: `git worktree remove Q:/fgl-2026-05-26 && git branch -d fgl/2026-05-26`.

## Risks

- Subagent stalls / runs over context budget. Mitigation: give each a tight file list
  (≤20 files), explicit "find P0/P1 only, skip nits", max-200-word findings cap.
- Findings overlap with prior reviews. Mitigation: tell each subagent to cite-novelty
  vs prior `AGENT_REVIEW_*.md` if material covered already.
- Synthesis becomes noise. Mitigation: cap synthesis at Top-20 P0/P1 items, P2 in appendix.
