# Adversarial Code Review — data-external subsystem

Reviewer: opus 4.7 (1M) /fgl audit, 2026-05-26 — incremental over 2026-05-24 baseline.
Scope (literal, per spec): 20 portfolio/* modules only. Findings in `data/crypto_data.py`
from the 2026-05-24 review are out of scope here and not re-cited.

---

## Top 3 (most impactful, sorted by silence × blast radius)

1. **`portfolio/microstructure_state.py:205-213`** — `persist_state()` writes the *computed* per-cycle state (ofi/ofi_zscore/spread_zscore) but DOES NOT persist the underlying `_snapshot_buffers`, `_spread_buffers`, or `_ofi_history` deques. After any loop restart (PF-DataLoop auto-restarts every logon + on crash, plus the 02:00 / 22:15 sleep schedule cycles), every ticker's rolling buffer resets to empty. `get_ofi_zscore` and `get_spread_zscore` then both return 0.0 until 10 new snapshots accumulate (~10 minutes at 60s cadence, longer at the 30-60s metals fast tick). The CLAUDE.md spec explicitly says "microstructure_state.py accumulates rolling OFI — must persist across loop restarts". This invariant is silently broken.
2. **`portfolio/crypto_precompute.py:154-235`** — Five raw `requests.get(...)` call sites (Binance spot 24hr, Binance FAPI premiumIndex, Binance FAPI openInterest, CoinGecko global, plus yfinance Ticker.history without `_yfinance_lock`) bypass `fetch_with_retry`, the binance rate limiter, the circuit breaker, AND the yfinance thread lock. No `raise_for_status` on the requests calls (only status==200 check, all other statuses silently zero out). On a Binance 429 burst this code keeps hammering until rate-limit banned, while the rest of the system thinks all sub-fetches "succeeded" with zeros.
3. **`portfolio/earnings_calendar.py:64,177`** — `_fetch_earnings_alpha_vantage` does `data = r.json()` directly with no check for the AV `{"Information": "..."}` rate-limit shape (only `_normalize_overview` in alpha_vantage.py checks `"Note"`, which is the *old* AV key). On rate-limit the AV body parses cleanly, `quarterlyEarnings=[]`, function returns None — then `get_earnings_proximity` caches `{"data": None, "time": now}` for 24h (line 177). Earnings gate is silently disabled for 24 hours after the first rate-limit hit, including on the actual earnings day.

## Critical (P0)

portfolio/microstructure_state.py:205-213: P0: `persist_state` writes only the computed summary, not the snapshot/spread/ofi-history deques. Buffers reset on every loop restart, OFI z-score returns 0.0 for ~10 min after restart. Spec violation. Fix: serialize the deques to disk on every persist and rehydrate in a `load_buffers()` helper called from metals_loop startup.
portfolio/crypto_precompute.py:159-167: P0: Binance 24hr ticker via raw `requests.get`, no rate limiter, no CB, no retry, no `raise_for_status`. Status 429/503 silently degrade to zeros. Fix: route through `binance_klines`-style helper or `fetch_json` with `_binance_limiter.wait()` and `binance_spot_cb`.
portfolio/crypto_precompute.py:178-197: P0: Same as above for `premiumIndex` and `openInterest` FAPI endpoints. Fix: use `portfolio.futures_data.get_open_interest` and add a funding-rate helper, both already wrap CB + limiter.
portfolio/crypto_precompute.py:166-167: P0: `float(d.get("lastPrice", 0)) or None` — converts an exact-zero or missing API value to `None`, which downstream treats as "fetch failed" and silently drops the data. If a real price came back as 0.0 (sentinel error), it's also dropped silently — both modes look identical. Fix: explicit `lastPrice = d.get("lastPrice"); out[key_price] = float(lastPrice) if lastPrice is not None else None`.
portfolio/crypto_precompute.py:210: P0: `yf.Ticker(sym).history(...)` called WITHOUT `_yfinance_lock`. Same thread-safety class as last review's P0-6 (which was fixed for `fetch_vix`). Concurrent yfinance via crypto precompute + sentiment workers corrupts shared SSL session. Fix: `from portfolio.shared_state import yfinance_lock; with yfinance_lock: hist = yf.Ticker(sym).history(...)`.
portfolio/earnings_calendar.py:64: P0: `data = r.json()` with no check for AV `"Information"` key (the newer rate-limit shape) or `"Note"`. Returns 200 OK + `{"Information":"..."}` → `quarterlyEarnings=[]` → returns None silently. Fix: `if isinstance(data, dict) and ("Note" in data or "Information" in data): logger.warning(...); return None`. [REPEAT of 2026-05-24 P1-7, still unfixed]
portfolio/earnings_calendar.py:176-177: P0: When fetch returns None (rate limit, AV outage, network), cache stores `{"data": None, "time": now}` and the 24h TTL check at line 170 returns the cached `None` for 24h. Earnings gate is silently disabled for an entire trading day after the first failure. Fix: only cache successful fetches; on failure use a separate short-TTL "negative cache" or skip caching entirely.
portfolio/alpha_vantage.py:150: P0: Only checks `"Note"` for AV rate-limit, not `"Information"` (current key as of late 2024+). Rate-limit responses pass through, `_normalize_overview` then returns None at line 99 because "Symbol" is missing → `_cb.record_failure()` at line 274 → circuit breaker accumulates failures on rate-limit responses, opens spuriously. Fix: add `"Information" in data` to the check. [REPEAT of 2026-05-24 P1-8, still unfixed]
portfolio/fx_rates.py:60-65: P0: Stale cache fallback returns `cached_rate` with NO upper bound. If `fetch_with_retry` fails for >2h, line 62 logs WARNING and fires Telegram, but **still returns the stale rate** with no `is_stale` flag. Silver trade in `metals_loop` multiplies SEK ladder by this stale rate; a 5% SEK/USD move during a multi-hour FX API outage produces 5% size error on every metals leg. Fix: either return `(rate, is_stale)` tuple so callers can guard, or raise after a hard cutoff (e.g. 6h) and let callers fall back to FX_RATE_FALLBACK with explicit knowledge.

## Important (P1)

portfolio/data_collector.py:222-229: P1: `yf.download(..., auto_adjust=True)` returns *split- and dividend-adjusted* close, not raw close. For MSTR (paid no dividends but had a 10:1 split Aug 2025) historical bars before the split silently shift. Cross-references between alpaca (raw close) and yfinance fallback (adjusted close) on the same ticker produce phantom 10x price differences. Fix: pass `auto_adjust=False` and use 'Close', OR pass `auto_adjust=True` and document the choice in CLAUDE.md.
portfolio/price_source.py:146-149: P1: Same `auto_adjust=True` issue as data_collector. CL=F/BZ=F have no dividends so the practical impact is zero today, but if MSTR ever falls into the alpaca → yfinance fallback path (line 235), it gets adjusted close while the rest of the system has been receiving raw close from Alpaca. Same fix.
portfolio/futures_data.py:61: P1: `float(data["openInterest"])` — bare key access. If Binance returns `{}` (rare error path) or schema changes, KeyError propagates uncaught into the worker. Fix: `float(data.get("openInterest") or 0)` and guard against None. [REPEAT of 2026-05-24 P2-18, still unfixed]
portfolio/crypto_macro_data.py:381-382: P1: `round(sum_7d, 1) if sum_7d else None` — sum_7d == 0.0 (perfectly balanced inflow/outflow) is falsy in Python, mis-classified as "no data". Fix: `... if sum_7d is not None else None`. [REPEAT of 2026-05-24 P1-10, still unfixed]
portfolio/crypto_macro_data.py:202-225: P1: `compute_gold_btc_ratio` reads BTC and Gold prices from `agent_summary_compact.json` (cached, potentially stale) instead of hitting live Binance. CLAUDE.md rule #3 ("Live prices first") violation. Fix: route through `portfolio.price_source.fetch_klines("BTC-USD", "1h", 2)` and same for XAU-USD. [REPEAT of 2026-05-24 P1-11, still unfixed]
portfolio/crypto_macro_data.py:108: P1: `now = datetime.date.today()` is *system-local*, not UTC. Deribit expiries are UTC; around midnight CET this picks wrong nearest expiry by 1 day for short-dated weeklies. Fix: `now = datetime.datetime.now(datetime.UTC).date()`. [REPEAT of 2026-05-24 P2-24, escalated to P1 — affects nearest-expiry option chain math]
portfolio/social_sentiment.py:32,65: P1: Raw `requests.get` (no `fetch_json`, no retry, no CB) plus errors go to `print(f"...")` at lines 110, 122 instead of `logger.warning`. When Reddit returns 429, this code retries on the next cycle with no backoff and gets IP-banned. Fix: migrate to `fetch_json(...)` and replace `print` with `logger.warning`. [REPEAT of 2026-05-24 P1-9, still unfixed]
portfolio/onchain_data.py:278-284: P1: When `_load_config_token()` returns None, function silently serves 2x-stale cache (24h instead of 12h) with only a DEBUG log. Operator with revoked/missing BGeometrics token gets day-old on-chain data with no surfaced warning. Fix: log WARNING + add `is_stale: True` field so the on-chain voter can abstain. [REPEAT of 2026-05-24 P2-20, still unfixed]
portfolio/sentiment.py:329-338: P1: 120s subprocess timeout in `_run_model` fallback path. Ticker pool has 500s timeout but multiple wedged sentiment subprocesses can chain to >300s. Especially when GPU lock is held by llama-server during model swap, the subprocess sits waiting indefinitely on a CPU model load. Fix: drop subprocess timeout to 30s. [REPEAT of 2026-05-24 P2-21, still unfixed]
portfolio/data_collector.py:296-300: P1: `"insufficient data ({rows} rows)"` log message is misleading — `compute_indicators(df)` can return None for many reasons (NaN saturation, schema break, indicator computation failure). Fix: change label to `"compute_indicators returned None ({rows} rows)"` so operators don't dismiss real schema-break incidents as "harmless slow start". [REPEAT of 2026-05-24 P1-12, still unfixed]
portfolio/fear_greed.py:152-154: P1: Belt-and-suspenders only handles 2D Close. If yfinance returns a scalar (degenerate single-row case for `^VIX`), `float(close_series.iloc[-1])` raises AttributeError; signal goes silently dead. Fix: wrap final `float()` in try/except with logger.warning. [REPEAT of 2026-05-24 P1-14, still unfixed]
portfolio/crypto_precompute.py:179-197: P1: Inner `try/except: pass` (lines 187, 196) swallows all errors with no logging. When FAPI flakes, operator sees nothing — funding rate and OI silently fall to None for ETH/BTC for 4h until the next precompute. Fix: at minimum `logger.debug("crypto_precompute %s fetch failed: %s", sym, e)`.
portfolio/data_collector.py:230-231: P1: yfinance `ValueError("No yfinance data for ...")` caught at `_fetch_one_timeframe` and logged at DEBUG only. When MSTR's Alpaca → yfinance fallback fails too, operator gets nothing. Fix: WARNING when yfinance is the primary fallback (closed-market path). [REPEAT of 2026-05-24 P1-15, still unfixed]

portfolio/microstructure.py:36-38: P2: `depth_imbalance` returns 0.0 when bid_vol OR ask_vol is <= 0. portfolio/microstructure.py:107-148: P2: `compute_ofi` reads `prev["best_bid"]` and `curr["best_bid"]` without checking they exist or are non-None. portfolio/onchain_data.py:282: P1→P2: Existing graceful-stale-cache path (no-token case) tags onto P2-20 — flagged again for visibility.

## Recommendations (priority order)

1. **Fix microstructure_state buffer persistence immediately** — single highest-impact latent issue. The OFI z-score signal feeds metals/orderbook_flow voter; every restart wipes its history and silently returns 0.0 z-scores for ~10 minutes. The metals loop restarts more often than that under failure recovery.
2. **Rewrite crypto_precompute to use `fetch_json` + existing wrappers** — five raw `requests.get` + thread-unsafe yfinance is the largest API-hygiene regression introduced since 2026-05-24. Use `portfolio.futures_data.get_open_interest`, add a `binance_24hr_ticker` helper to `data_collector`, route yfinance via `_yfinance_lock`.
3. **Earnings calendar: fix the silent 24h cache-on-failure + missing `"Information"` check** — gate misfires on earnings days are real money risk.
4. **FX rate: surface staleness to callers** — SEK conversion silently drifts during long FX outages; expose `is_stale` flag or hard-cutoff.
5. **Cross-fix Alpha Vantage `"Information"` key in both `alpha_vantage.py:150` AND `earnings_calendar.py:64`** — same root cause, both unfixed since 2026-05-24.
6. **Decide `auto_adjust` policy globally** — both data_collector.py and price_source.py default to `True`; standardize and document, or set to `False` so close-of-bar matches Alpaca/Binance raw closes.

External-API code has matured well around outright crashes (circuit breakers, retries, rate limiters are in place where used). The dominant residual risk remains **silent semantic degradation** — bypassed limiters in crypto_precompute, stale data without freshness flags (fx_rates, onchain), wrong dict keys for newer API rate-limit responses (AV `"Information"`), and the unpersisted microstructure rolling buffers that violate the explicit spec invariant.
